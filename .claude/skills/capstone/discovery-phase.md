# Discovery Phase - Problem & Market Validation

## Purpose

Execute Discovery phase activities for the AI Capstone: validate that the AGI pill addresses a real, valuable problem worth solving.

## Context

Reference: `@context:agents/_shared/context`

**Phase Goal**: Identify and validate a compelling AI product opportunity before PRD deadline (Feb 15).

## Inputs Required

- **Research focus**: What aspect of discovery to work on
- **Existing data**: Any research, interviews, or market data collected
- **Time constraints**: Days until PRD deadline

## Discovery Phase Activities

### 1. Problem Exploration
- Define initial problem hypotheses
- Identify target user segments
- Map current alternatives and workarounds
- Document assumptions to validate

### 2. Market Research
- Research AGI pill market opportunities
- Define market size (TAM/SAM/SOM)
- Analyze competitive landscape
- Identify market trends and timing
- Assess barriers to entry

### 3. Use Case Development
- Generate potential use cases for AGI pill
- Evaluate each use case:
  - Market opportunity
  - Technical feasibility
  - Differentiation potential
  - Alignment with AI capabilities
- Prioritize using RICE or similar framework
- Create use case cards with validation criteria

### 4. User Research
- Plan and conduct user interviews (target: 10+)
- Create interview guide focused on problems
- Synthesize interview findings
- Create user personas
- Map jobs-to-be-done

### 5. Problem Validation
- Define validation criteria for each hypothesis
- Test problem hypotheses with evidence
- Document validated vs. invalidated assumptions
- Synthesize into clear problem statement

## Instructions

### Quick Start (Given Tight Timeline)

1. **Day 1-2: Market Scan**
   - Research 3-5 AGI pill opportunity areas
   - Identify initial use case candidates
   - Map competitive landscape

2. **Day 3-5: Rapid User Research**
   - Conduct 5-10 quick user interviews
   - Focus on problem validation, not solution
   - Synthesize key pain points

3. **Day 6-8: Use Case Selection**
   - Evaluate use cases against criteria
   - Select primary use case
   - Document validation evidence

4. **Day 9-10: Problem Statement**
   - Finalize validated problem statement
   - Prepare for Design phase transition
   - Brief mentor (if sync scheduled)

### Full Discovery Process

1. **Assess Current State**
   - Review capstone context for phase progress
   - Check what discovery work has been done
   - Identify priority activity based on timeline

2. **Execute Discovery Activity**
   - Based on request, execute appropriate activity
   - Use Jobs-to-be-Done framework for use cases
   - Apply lean validation principles

3. **Document Learning**
   - Capture key insights
   - Update use case tracking table
   - Flag items for mentor discussion

4. **Assess Phase Readiness**
   - Check exit criteria progress
   - Recommend next steps or phase transition

## Use Case Evaluation Framework

For each potential use case, assess:

| Criterion | Weight | Score (1-5) | Notes |
|-----------|--------|-------------|-------|
| Problem severity | 25% | | How painful is this problem? |
| Market size | 20% | | How large is the opportunity? |
| AI advantage | 20% | | Does AI uniquely solve this? |
| Feasibility | 20% | | Can we build this in time? |
| Differentiation | 15% | | How crowded is the market? |

**Total Score**: Weighted average
