# Deploy Phase - Launch & Presentation Readiness

## Purpose

Execute Deploy phase activities for the AI Capstone: prepare for final submission, define success metrics, and complete all capstone deliverables.

## Context

Reference: `@context:agents/_shared/context`

**Phase Goal**: Complete final presentation and submission by April 15, 2026.

## Inputs Required

- **Deployment focus**: GTM planning, metrics definition, or presentation prep
- **Built product**: Working prototype from Develop phase
- **Audience**: Mentor, evaluators, potential users/investors

## Deploy Phase Activities

### 1. Go-to-Market Planning

- Define target launch audience
- Create positioning and messaging
- Plan distribution channels
- Set launch timeline
- Identify early adopter strategy

### 2. Success Metrics Definition

- Define key metrics (North Star, supporting, guardrails, trade-off)
- Set targets and thresholds
- Plan measurement approach
- Create tracking dashboard concept
- Connect metrics to original problem

### 3. Presentation Preparation

- Create capstone presentation structure
- Develop compelling narrative arc
- Prepare investor/pitch deck format
- Plan demo integration
- Practice and refine delivery

### 4. Stakeholder Alignment

- Final mentor review with Migdad
- Address evaluator criteria
- Gather endorsements/testimonials
- Document lessons learned

### 5. Capstone Completion

- Compile all deliverables
- Ensure documentation is complete
- Create retrospective
- Plan post-capstone next steps

## Presentation Structure

```markdown
## Capstone Presentation Outline

### 1. Hook (1-2 min)

- Compelling problem statement
- Real user story or data point
- Why this matters now

### 2. Problem & Opportunity (3-4 min)

- Validated problem (from Discovery)
- Market opportunity (TAM/SAM/SOM)
- Why existing solutions fail
- Target user persona

### 3. Solution (4-5 min)

- Product vision and approach
- Why AI/AGI is essential
- Key features and user flows
- Live demo or video walkthrough

### 4. Technical Approach (2-3 min)

- Architecture overview
- AI/ML components
- Key technical decisions
- Build vs. buy choices

### 5. Validation & Results (2-3 min)

- User research findings
- Usability testing results
- Key metrics (if available)
- User feedback and quotes

### 6. Go-to-Market (2 min)

- Target market and positioning
- Launch strategy
- Success metrics

### 7. Journey & Learnings (2 min)

- 4D process experience
- Key pivots or decisions
- What you'd do differently
- Mentor acknowledgment

### 8. Future Vision (1-2 min)

- Roadmap beyond MVP
- Scaling potential
- Call to action

### Total: 18-22 minutes
```

## Instructions

1. **Validate Readiness**
   - Ensure MVP meets minimum quality bar
   - Confirm core user value is demonstrable
   - Review any outstanding issues

2. **Execute Deploy Activity**
   - Use `/stakeholder-update` patterns for communications
   - Apply pitch/presentation best practices
   - Define measurable success criteria

3. **Prepare Deliverables**
   - Compile capstone documentation
   - Create compelling narrative
   - Highlight AI/AGI innovation

4. **Close the Loop**
   - Connect back to original problem validation
   - Show journey from Discovery to Deploy
   - Document key learnings
   - Thank mentor and supporters
